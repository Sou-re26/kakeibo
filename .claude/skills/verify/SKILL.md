# Verify (kakeibo)

## 1. 静的検証(必須)

```bash
npm run check   # expo lint + tsc --noEmit
```

両方成功して初めて次へ。失敗したら出力ごと報告する。

## 2. 起動

```bash
npx expo start        # Metro起動。Androidエミュレータは 'a' キーまたは npm run android
```

- 起動時に `useMigrations` がマイグレーションを自動適用する。マイグレーション追加を含む変更では、起動直後にエラー画面/ローディングのままにならないことを最初に確認する。
- DBは端末内の `kakeibo.db`。スキーマ変更のテストでデータをリセットしたい場合はアプリのアンインストール(=DB削除)を行うが、**実行前にユーザーへ確認する**。

## 3. 基本スモークフロー(引数なしの場合)

1. ホームタブが表示される(今月の収支が出る)
2. 入力画面: 金額を入れて種別(支出/収入/振替)を選び「次へ」
3. 詳細画面: カテゴリ選択 → 日付・店舗・メモを入力 → 保存
4. 保存後に `(tabs)` へ戻る
5. 履歴タブ: 保存した取引が表示される(既知バグ: 再フォーカスで更新されない。修正前は再起動して確認)
6. ダークモードでも同じフローを目視する(既知バグ: 履歴カードが読めない)

変更した画面がこのフロー外なら、その画面への到達手順と確認観点を報告に書く。

## 4. 注意

- 日付選択は現状Android専用(`DateTimePickerAndroid`)。iOS/Webでは日付機能の確認はできない。
- エミュレータが使えない環境では、静的検証の結果+上記フローの該当ステップを「未検証」として報告し、確認手順を残す。
